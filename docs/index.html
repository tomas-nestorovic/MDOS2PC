Changelog

1.4.3 RC2 (May 12, 2018)

  • added dialog to manually adjust the disk/image recognition sequence (i.e., order of DOSes that get crack on the image), under Image → Recognition
  • added possibility to recover also erroneous sector ID during dumping
  • changed determination of free space on a TRDOS image/disk (now based on the First empty sector information from the boot, instead of on information from directory; new approach allows to tweak the file system)
  • added pressing Enter key in the hexa-editor passes the focus to its previous owner
  • added support to export ZX Spectrum files with characters that are forbidden in FAT32 and higher (e.g., "Name%AT <> STOP ;o).P"); some limitations still hold
  • fixed importing of TRDOS files (first, when they lack export information, and second, physical addressing of sectors)
  • removed possibility to accept some i/o errors (e.g., data CRC) when dumping to a raw image; only Retry and almighty Abort are offered
  • added support for high-DPI screens; everything but PropertyGrid should now be laid out well regardless the Windows zoom factor

1.4.3 RC1 (Apr 20, 2018)

  • modified the disk/image recognition sequence (swapped the positions of TRDOS and GDOS)
  • fixed physical numbering of TRDOS sectors

1.4.2 (Apr 2, 2018)

  • added "context help" (effectively just links to on-line FAQs available through the "Need help?" hyperlink in the left lower corner of selected dialogs)
  • added the Allow zero-length files switch for GDOS files (to tweak the file system)
  • added suggestion to open the FAQs when the application is first launched
  • fixed track verification error message (stack-allocated buffer overflow)
  • updated on-line FAQs

1.4.1 (Mar 24, 2018)

  • fixed inclusion of "skipped" sectors to disk statistics, under Track Map → Helpers → Statistics
  • fixed interruption of automatic internal floppy drive latency determination (using the Cancel button)
  • fixed MS-DOS long file names removal
  • fixed filling of MS-DOS empty directory entries (and recurrently in subdirectories)
  • improved filling of MDOS, TRDOS, and GDOS empty directory entries
  • fixed marking of MS-DOS sectors as "dirty"
  • fixed importing of zero-length GDOS files (to tweak the file system)
  • fixed closing of the application when only the ZX Spectrum tape is modified
  • fixed importing of files between various ZX platforms
  • fixed addressing of GDOS sectors when the format is not 80x10 (e.g., is 27x10)
  • fixed MS-DOS disk/image initialization
  • added question to dump cylinders that are beyond the official format into a raw image
  • added possibility to set default MDOS file attributes if imported files have no explicit export information
  • updated on-line FAQs
  • changed name to "Real and Imaginary Disk Editor"

1.4 (Feb 2, 2018)

  • improved scheme on reporting file size
  • added support for MGT images
  • added GDOS implementation (but UniDOS is currently not supported)
  • improved ZX Spectrum file export information (motivation: Bobby Carrot)
  • fixed deletion in hexa-editor if content to delete selected "from end towards beginning"
  • fixed opening of read-only images (e.g., from a CD-ROM)
  • fixed marking of TRDOS sectors as "dirty"
  • fixed the ellipsis button in PropertyGrid

1.3.7 (Oct 22, 2017)

  • fixed deletion of MS-DOS directories that occupy two or more clusters
  • improved TRDOS and MS-DOS robustness if their key sectors not found
  • added Motor off and Prefer relative seeking parameters the the internal floppy setting dialog (see why)
  • added the File Manager → Selection → Properties command
  • added the possibility to ignore "dot" and "dotdot" directory entries in MS-DOS
  • added the Image → Patch command to "patch" an existing image (see how and why)
  • added on-line testing of application recency (Help → Check for Updates)
  • improved usability on Windows 10 (Windows 8.x still untested)
  • other minor improvements

1.3.6 (Sep 16, 2017)

  • fixed the stepping speed for MDOS drives
  • fixed MS-DOS functionality
  • improved the formatting of raw image (faster)
  • improved the formatting sequence of internal floppy drive
  • added reporting on bugs during dumping (as HTML page)
  • added possibility to recover bad sector data (by either correcting the CRC or replacing the bad content with substitute data)
  • added support for reproduction of these i/o floppy errors: End of Cylinder, Missing Address Mark, and Missing DAM
  • added possibility to verify data written to the disk/image, including i/o errors reproduction
  • removed manual input of first sector number on a track in Format Cylinder dialog (can always be inferred automatically)
  • other minor improvements

1.3.5 (Jun 14, 2017)

  • fixed marking of raw image sectors as "dirty"
  • improved unformatting of DSK image tracks (just cosmetic affairs, not functional)
  • added automatic determination of internal floppy drive latencies (first experimental version)
  • added basic web pages available through the Help menu

1.3.4 (May 20, 2017)

  • fixed DSK image implementation (motivation: Arctic Fox, DB1 Plus Three Disc Backup Utility, Discology +3, Gemini Wing, and many others)
  • fixed the maximum number of ZX Spectrum files to 2048 (motivation: Astro Marine Corps)
  • improved ZX Spectrum file export information to now contain the "universal type of file" (useful when porting files between ZX platforms)
  • improved the formatting sequence of internal floppy drive
  • added displaying of bad sector data in Track Map
  • added dialog on internal floppy drive settings
  • added support for reproduction of these i/o floppy errors: Deleted DAM, Data Field CRC Error, and Error in ID or Data Field
  • added support for duplicate sector IDs (motivation: The Untouchables)
  • added support for reproduction of 8kB long sectors on 2DD floppies (motivation: Extreme, The New Zealand Story)
  • other minor improvements

1.3.3 (Mar 31, 2017)

  • fixed "Error 183" when rewriting an existing image

  • fixed TRDOS disk defragmentation
  • fixed the Image → Dump action to correctly recognize and process single-sided images
  • improved the Compare Files dialog visuals in Track Map
  • added support for "standard" DSK images (so far only "Revision 5" has been supported)
